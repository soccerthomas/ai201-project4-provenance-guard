# AI Provenance Guard

A Flask API that analyzes text and returns a label indicating whether it was likely written by a human or generated by AI.

---

## Architecture Overview

1. **POST /submit** receives `text` and `creator_id`
2. **Groq LLM** scores the text from 0.0 (human) to 1.0 (AI)
3. **Stylometric analysis** computes sentence length, vocabulary diversity, and rhythm variance into a second score
4. **Confidence scoring** combines both signals into a weighted score
5. **Label generation** maps the score to one of three labels
6. **Audit log** records everything to `logs/audit.json`
7. **JSON response** is returned to the caller

Appeals go to **POST /appeal** — the submission status flips to `pending_review` and the appeal is logged.

```
POST /submit → Groq Signal → Stylometric Signal → Confidence Score → Label → Audit Log → Response
POST /appeal → Status Update → Audit Log → Response
```

---

## Detection Signals

### Signal 1: Groq LLM Assessment
Sends the text to `llama-3.3-70b-versatile` and asks it to score how likely the text is AI-generated. Returns a 0.0–1.0 score.

**Why:** An LLM picks up on phrasing patterns and structural habits that simple heuristics miss.

**Blind spot:** Polished human writing can look AI-generated. Heavily edited AI output can look human.

### Signal 2: Stylometric Heuristics
Computes three properties of the text:
- **Average sentence length** — AI tends to cluster in the 15–25 word range
- **Type-token ratio** — lower vocabulary diversity is more AI-like
- **Burstiness** — humans write with more irregular sentence rhythm than AI

**Why:** It's transparent and deterministic — you can see exactly what it's measuring.

**Blind spot:** Poetry and technical writing naturally score as AI-like. Very short text returns a neutral 0.5.

---

## Confidence Scoring

```
Confidence = (0.6 × Groq Score) + (0.4 × Heuristic Score)
```

Groq gets more weight because it captures broader patterns the heuristics can't.

| Confidence  | Attribution  |
|-------------|--------------|
| 0.00 – 0.39 | likely_human |
| 0.40 – 0.69 | uncertain    |
| 0.70 – 1.00 | likely_ai    |

### Example Submissions

**High confidence AI** — formal, uniform paragraph:
```json
{ "llm_score": 0.8, "heuristic_score": 0.5575, "confidence": 0.703, "attribution": "likely_ai" }
```

**Low confidence / likely human** — casual, irregular writing:
```json
{ "llm_score": 0.21, "heuristic_score": 0.3795, "confidence": 0.2778, "attribution": "likely_human" }
```

## Transparency Labels

### Likely AI Generated
```
Likely AI Generated
Confidence: XX%
Our analysis found strong indicators commonly associated with AI-generated writing.
This result is probabilistic and should not be treated as definitive proof.
```

### Likely Human Written
```
Likely Human Written
Confidence: XX%
Our analysis found stronger evidence of human writing patterns than AI-generated patterns.
This result is probabilistic and not a guarantee.
```

### Uncertain
```
Uncertain
Confidence: XX%
The available evidence is mixed. The system cannot confidently determine whether
this content was written by a human or generated by AI.
```

## Rate Limiting

- 10 requests per minute per IP
- 100 requests per day per IP

The per-minute limit prevents rapid automated probing. The per-day limit is generous enough for normal use while blocking large-scale scraping.

---

## Known Limitations

**Formal human writing scores as AI.** The stylometric signal penalizes uniform sentence length and low vocabulary variance — but those are also features of academic or professional writing. A policy memo written by a human will look AI-generated to the heuristic. This is a property of the signal itself, not a data problem.

**Short text is unreliable.** Submissions under two sentences don't give the stylometric module enough to work with. These return a neutral 0.5 heuristic score and almost always land in "Uncertain."

## Spec Reflection

**What helped:** Writing the three label variants in `planning.md` before any code made `labels.py` straightforward to implement — the strings were already decided.

**What diverged:** The spec described SQLite for the audit log. I used a JSON file instead because it was easier to inspect during development. For a real deployment, SQLite would be the right call since concurrent writes to a JSON file aren't safe.

## AI Usage

**Instance 1:** I gave Claude the detection signals section and architecture diagram and asked it to generate the Flask skeleton and Groq signal function. The Groq prompt it wrote returned freeform text instead of a clean score — I revised it to explicitly request a single decimal between 0 and 1.

**Instance 2:** I asked Claude to generate the stylometric module and confidence scoring logic. The scoring function it produced used a binary split at 0.5 instead of the three-tier thresholds from my spec. I caught this during testing and corrected `get_attribution` to match the planned ranges.
